## 문제 상황
- input type range를 이용해 슬라이더를 표현하는 상황에서 가상 요소를 통해 thumb를 대체하려고 한 상황

Firefox에서 커스텀 thumb이 노출되지 않는 이유를 분석하겠습니다.

🔍 문제 원인
`::after` 가상 요소가 `<input>` 요소에서 동작하지 않음
Firefox를 포함한 대부분의 브라우저에서 `<input>` 요소는 ::before와 ::after 가상 요소를 지원하지 않습니다.

이유:

input은 replaced element(대체 요소)로, 내부 구조를 CSS로 제어할 수 없습니다.
Chrome/Safari는 일부 상황에서 예외적으로 동작할 수 있지만, Firefox는 엄격하게 표준을 따릅니다.
2. ::-moz-range-thumb이 transparent로 설정됨
Chrome은 ::after로 커스텀 thumb을 그릴 수 있지만
Firefox는 ::after가 동작하지 않으므로 ::-moz-range-thumb만 보임
하지만 이것도 transparent로 설정되어 있어 아무것도 보이지 않음
✅ 해결 방법
Option 1: Wrapper 요소 추가 (권장)
input 요소를 감싸는 wrapper에서 ::after를 사용합니다.
