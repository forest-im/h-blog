# 글쓰기 규칙

이 블로그에 글을 쓸 때 따르는 규칙.

## 역할

- **TIL** = AI로 초안 작성 가능.
- **blog** = Hazel이 직접 작성. AI는 blog 글을 쓰지 않음.

## 대원칙

- 목적은 **잘 쓴 글이 아니라 배운 걸 남기는 것**.
- **draft = 발행.** 짧아도, 거칠어도 올림.
- 완성도를 기준으로 삼지 않음.

## 문체 (공통)

- **음슴체(`~음` / `~함` / `~됨` / `~임`)로 끝냄.**
- **평서문(`~다`), 존댓말(`~습니다`) 금지.**
- 과거는 `~었음` / `~았음`, 현재는 `~음` / `~됨`.
- 예: "안 지워진다" ✕ → "안 지워짐" ○
- frontmatter의 `summary`도 음슴체로.

## TIL — `content/til/`

- **언제**: 배운점 하나를 짧게 남길 때.
- **형식**: 개조식. 문장 최소, 키워드·화살표(`→`) 위주.
- **제목**: 핵심 한 줄. `TIL:` 같은 접두어 붙이지 않음 (메뉴가 이미 TIL임).
- **섹션**: 상황에 맞게. 고정 아님. (자주 쓰는 틀: 문제 / 원인 / 확인 / 해결)
- 서사·감정선 넣지 않음. 요점만.

```md
## 원인
- 강력 새로고침 = HTTP 캐시만
- SW/CacheStorage → 안 지워짐
- 시크릿 = SW 없음 → 정상
```

## blog — `content/blog/`

- **언제**: 뿌리까지 파거나, 길게 정리하고 싶을 때.
- **형식**: 문단 서술(개조식 아님). 단 어미는 음슴체.
- **흐름**: 증상 → 표면 원인 → 근본 원인 → 해결 → 교훈.
- 1인칭 서사·과정은 허용 (예: "처음엔 캐시를 의심했음").
- 코드·구조는 코드블록으로.

## 공통 형식

- frontmatter: `title` / `date`(YYYY-MM-DD) / `summary`.
- 파일 위치가 곧 메뉴: `content/til/` 또는 `content/blog/`.
- 파일명은 kebab-case.

## 로컬 드래프트

- 파일명이 `_`로 시작하면 **로컬 전용 드래프트** — git에 커밋되지 않고 로컬 dev에서만 보임.
- 발행하려면 `_` 접두어를 제거.
